1) What is an Algorithm?

Posted on 2025.09.18

An algorithm is a finite sequence of well-defined steps that transforms input into output.

  • Finiteness: Must terminate after finite steps.
  • Definiteness: Each step is precise and unambiguous.
  • Input & Output: Maps given inputs to expected outputs.

Examples: Sorting, searching, cryptography.

2) Why Efficiency Matters

Different algorithms solve the same problem with radically different performance.

  • Bubble Sort: \(O(n^2)\). Sorting 1,000 elements → ~1,000,000 operations.
  • Quick Sort: \(O(n\log n)\). Sorting 1,000 elements → ~10,000 operations.
Scaling up: At millions or billions of data points, efficiency means the difference between seconds and hours.

3) Complexity Analysis

  • Time Complexity: Growth of execution time as function of input size \(n\).
  • Space Complexity: Extra memory needed beyond input storage.

We focus on asymptotic analysis: behavior as \(n \to \infty\).

4) Big-O Notation

Big-O describes an upper bound on growth rate.

  • \(O(1)\): constant time (array indexing).
  • \(O(\log n)\): logarithmic (binary search).
  • \(O(n)\): linear (linear scan).
  • \(O(n\log n)\): log-linear (merge/quick sort).
  • \(O(n^2)\): quadratic (nested loops, bubble sort).
  • \(O(2^n)\): exponential (subset enumeration).
  • \(O(n!)\): factorial (Traveling Salesman brute force).

Formal definition: \(f(n) \in O(g(n))\) if ∃ constants \(c,n_0>0\) such that \(|f(n)| \leq c \cdot g(n)\) for all \(n \geq n_0\).

5) Big-O, Big-Ω, Big-Θ

  • Big-O (O): Worst-case upper bound.
  • Big-Ω (Ω): Best-case lower bound.
  • Big-Θ (Θ): Tight bound (both upper and lower).

Example: Insertion sort → Worst: \(O(n^2)\), Best: \(\Omega(n)\), Average: \(\Theta(n^2)\).

6) Sorting Examples

Bubble Sort

  • Simple but inefficient: \(O(n^2)\).
  • Good for teaching, rarely used in practice.

Merge Sort

  • Divide-and-conquer.
  • Guaranteed \(O(n\log n)\).
  • Stable, good for large datasets.

Quick Sort

  • Average: \(O(n\log n)\).
  • Worst-case: \(O(n^2)\) (rare with randomization).
  • In practice, one of the fastest.

7) Applications in Computer Science

  • Databases: Search efficiency (B-Trees, Hash Tables).
  • Cryptography: Hardness of problems (e.g., factoring \(O(e^{n^{1/3}})\)).
  • AI: Graph search (A*, Dijkstra).
  • Networking: Routing protocol optimization.
  • Software Engineering: Code review: "What’s the time complexity?"

8) Reflection

Algorithm analysis with Big-O is the mathematical language for efficiency. Beyond correctness, scalability is the true measure of quality.

Insight: Efficiency is not a luxury but the essence of computer science. Mastering algorithm design and complexity analysis is essential across fields — security, AI, systems, or networks.
